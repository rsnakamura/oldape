Run Iperf Traffic to DUT¶

Related Requirements¶

  • A.6. System impels iperf traffic to the Device Under Test (DUT).
  • A.8. System saves iperf data for both Device Under Test (DUT) and Traffic PC (TPC).

Preconditions¶

  • The Device Under Test (DUT) is connected to the Access Point (AP)
  • The DUT and Traffic PC (TPC) are on the same subnet
  • TPC and DUT have iperf installed on their PATH

Postcondition¶

The data from the iperf Session is stored in Files.

Failed End Condition¶

No, partial, or corrupted data has been saved for the TPC.

Trigger¶

All-In-One Performance Evaluation Tools (APETools) requests iperf to DUT test.

Main Path¶

  1. System starts an iperf server on the DUT.
  2. System waits for the server to start.
  3. System imples the TPC to send traffic to the DUT.
  4. System saves the standard output of the DUT and TPC as it is generated during test.

Alternate Paths¶

2.1. System is unable to start the iperf server.

2.1.1. CommandError or ConnectionError is rased based on cause.

2.1.2. The test exits.

4.1. System is unable to impel the TPC to send traffic.

4.1.1. A CommandError, ConfigurationError, or ConnectionError is raised based on cause.

4.1.2 The test exits.
