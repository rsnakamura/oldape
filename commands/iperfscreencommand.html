The Iperf Screen Command¶

This is a more generalized iPad hack.

Problem¶

  • The iPad only has a WiFi (command-line) interface
  • The requirement is to reserve WiFi traffic to testing (while measuring iperf throughput)
  • The output of the UDP server has to be preserved (as well as the client-side for TCP)

A Possible Sequence¶

  1. Open an SSH Connection to the iPad (via its wireless interface)

  2. Use top to find any existing iperf sessions

  3. Use kill to kill any iperf sessions found

  4. Run the iperf command on the iPad, re-redirecting the output to a file

  5. Close the ssh-connection?

    • If it’s run in screen this doesn’t seem necessary
    • Can you just redirect the daemon output and close the connection?

  6. Run iperf traffic

  7. Open the connection to the iPad

  8. Use top to find any iperf sessions on the iPad

  9. Use kill to kill any iperf sessions found

  10. Use scp to copy the output of the iperf session to the control PC
