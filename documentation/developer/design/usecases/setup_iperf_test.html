Setup Iperf Test Use Case¶

Related Requirements¶

  • A.3. System communicates with the Device Under Test (DUT) via Android Debug Bridge (ADB).
  • A.4. System communicates with the Traffic PC (TPC) via Secure Shell (SSH).
  • A.5. System kills iperf sessions on the Device Under Test (DUT) and Traffic PC (TPC).

Preconditions¶

  • The Control PC (CPC) is connected to the DUT and communicating via ADB
  • The Traffic PC (TPC) has a running SSH Server
  • CPC and TPC are on the same subnet

Postcondition¶

DUT and TPC have no running iperf processes.

Trigger¶

All-In-One Performance Evaluation Tools (APETools) requests throughput test.

Main Path¶

  1. The system establishes connection to DUT and TPC
  2. The system kills all existing iperf processes on the TPC and DUT.

Alternate Paths¶

2.1. The system is unable to connect to the DUT or TPC.

2.1.1. A ConnectionError or ConfigurationError is raised based on cause. 2.1.2. The test exits.

3.1. Running iperf processes remain on the DUT or TPC.

3.1.1. A CommandError is raised. 3.1.2. The test exits.
