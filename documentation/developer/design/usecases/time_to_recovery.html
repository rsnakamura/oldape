Time to Recovery Use Case¶

The system impels the Device Under Test (DUT) to ping the Traffic PC (TPC) until it succeeds, recording the time of the first success.

Related Requirements¶

  • A.14 System waits for Device Under Test (DUT) to have network connection to the Traffic PC (TPC).
  • A.15 System impels Device Under Test (DUT) to ping Traffic PC (TPC).

Preconditions¶

  1. The Device Under Test (DUT) and Traffic PC (TPC) are on the same subnet.
  2. The Device Under Test (DUT) is attached to the Control PC (CPC) via Android Debug Bridge (ADB).
  3. The Device Under Test (DUT) is in a recovering or ready state.

Postcondition¶

Time to first ping is returned calculated.

Main Path¶

  1. DUT pings TPC
  2. If ping and is first ping, set first ping time.
  3. If ping increment ping count
  4. If not ping, set first ping time to none and ping count to 0.
  5. If ping count matches threshold, return the time to the first ping.
  6. Go to step 1.
