Throughput Over Time (TOT) Requirements¶

A. Kernel Requirements¶

  1. A.1. System interprets Configuration Files.
  2. A.2. System is invoked via the command line.
  3. A.3. System communicates with the Device Under Test (DUT) via Android Debug Bridge (ADB).
  4. A.4. System communicates with the Traffic PC (TPC) via Secure Shell (SSH).
  5. A.5. System kills iperf sessions on the Device Under Test (DUT) and Traffic PC (TPC).
  6. A.6. System impels iperf traffic to the Device Under Test (DUT).
  7. A.7. System impels iperf traffic from the Device Under Test (DUT).
  8. A.8. System saves iperf data for both Device Under Test (DUT) and Traffic PC (TPC).
  9. A.9. System copies its log file to the data folder.
  10. A.10 System copies the configuration file to the data folder.
  11. A.11 System runs throughput tests.
  12. A.12 System uses Naxxx environmental affector to change Access Point (AP).
  13. A.13 System captures device log output and sends it to storage.
  14. A.14 System waits for Device Under Test (DUT) to have network connection to the Traffic PC (TPC).
  15. A.15 System impels Device Under Test (DUT) to ping Traffic PC (TPC).
