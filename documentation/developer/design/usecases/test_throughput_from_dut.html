Test Throughput From DUT Use Case¶

Related Requirements¶

  • A.1. System interprets Configuration Files.
  • A.2. System is invoked via the command line.
  • A.3. System communicates with the Device Under Test (DUT) via Android Debug Bridge (ADB).
  • A.4. System communicates with the Traffic PC (TPC) via Secure Shell (SSH).
  • A.5. System kills iperf sessions on the Device Under Test (DUT) and Traffic PC (TPC).
  • A.7. System impels iperf traffic from the Device Under Test (DUT).
  • A.8. System saves iperf data for both Device Under Test (DUT) and Traffic PC (TPC).

Preconditions¶

  • The APE has setup a well-formed configuration file
  • The Control PC (CPC) is connected to the DUT and communicating via ADB
  • The Traffic PC (TPC) has a running SSH Server
  • The Device Under Test (DUT) is connected to the Access Point (AP)
  • CPC and TPC are on the same subnet
  • The DUT and Traffic PC (TPC) are on the same subnet
  • TPC and DUT have iperf installed on their PATH

Postcondition¶

The data from the iperf Session is stored in files.

Failed End Condition¶

No, partial, or corrupted data has been saved for the DUT.

Trigger¶

All-In-One Performance Evaluation Tools (APETools) requests iperf to DUT test.

Main Path¶

  1. (:include: Setup Session) The test components are set up.
  2. (:include: Setup Iperf Test) Iperf Test Preparation completed.
  3. (:inlclude: Run Iperf Traffic From DUT) Iperf Traffic from DUT is run.
  4. (:include: Cleanup Session) Log and Configuration files are copied to the data folders.

Alternate Paths¶

1.1. System is unable to run Iperf Test Preparation.

1.1.1. Test exits.

2.1. System is unable to run Traffic from the DUT to the TPC.

2.1.1. Test exits.
